# 安全与合规披露

## 漏洞报告

若你发现本仓库代码存在 **可利用的安全漏洞**（例如远程代码执行、路径穿越、敏感信息泄露等），请 **不要** 在公开 Issue 中直接披露利用细节。

建议做法：

1. 使用 GitHub **Private vulnerability reporting**（若仓库已开启），或  
2. 通过仓库维护者在 Profile 中公开的联系方式进行 **私信** 说明（请勿在 Issue 中粘贴密钥或账号）。

维护者会在合理时间内确认并协调修复与公开说明（CVE 与否由严重程度决定）。

## 版权与合规联系

若你认为本仓库内容：

- 侵犯著作权、商标权或其他知识产权；  
- 违反你所在司法辖区的法律；  
- 或违反某产品服务条款且应下架；

请通过 **私信** 或平台提供的版权/滥用举报渠道联系，并说明：

- 权利依据与涉及内容（文件路径或提交哈希）；  
- 你希望采取的措施（删除、修改声明等）。

我们无意传播破解工具或盗版资源；在核实合理请求后会尽快处理。

## 研究边界说明

本项目的研究背景已在 [README.md](README.md) **「研究与溯源声明」** 中说明。本仓库 **不接受** 以「贡献」名义提交的破解补丁、 stolen 二进制或用于非法用途的利用代码。
